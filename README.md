# ValidMLInference (Stata port)

A Stata translation of the Python package [**ValidMLInference**](https://github.com/timothymchristensen/ValidMLInference) (Kurczynski & Christensen, MIT-licensed). It implements the bias-correction and joint-estimation methods for regressions whose covariates are generated by AI or machine learning developed in:

- Battaglia, L., Christensen, T., Hansen, S., & Sacher, S. (2025). *Inference for Regression with Variables Generated by AI or Machine Learning*. [arXiv:2402.15585](https://arxiv.org/abs/2402.15585).
- Christensen, T., & Hansen, S. (2025). *Performing Valid Inference with AI/ML-Generated Covariates: A Guide for Empirical Practice* (Papers and Proceedings).

This port is **not affiliated with the upstream authors.** Please cite the papers above for the methods. Bug reports about the Stata implementation should go to this repository's issue tracker, not to the upstream Python repo.

**Status:** v0.6.0, 2026-05-26. All seven public commands (`ols`, `ols_bca`, `ols_bcm`, `ols_bca_topic`, `ols_bcm_topic`, `one_step`, `one_step_gmm`) are implemented and pass Python-parity on the shipped fixture suite (11/11 cases). `one_step` accepts a closed `dist(normal|laplace|t)` menu — the Stata-side equivalent of Python's `distribution=` callback. Help files (`.sthlp`) are in place for every command. The package installs via `net install` (see below).

## Requirements

- Stata 17 or later.

The Stata package itself has no runtime dependencies beyond Stata. Python is required only to regenerate parity-test fixtures from the original `ValidMLInference` package.

## Installation

### From GitHub (recommended)

```stata
net install validmlinference, ///
    from("https://raw.githubusercontent.com/lranaldi/validmlinference-stata/main/") ///
    replace
```

> Adjust the URL to whatever GitHub user / repo / branch you push to. The `from()` location must contain `stata.toc` and `validmlinference.pkg` at its root.

This downloads the seven `.ado` files, the compiled Mata library `lvmli.mlib`, and the seven `.sthlp` help files into Stata's `PLUS` directory. No further setup is needed — the commands are then available like any other Stata command:

```stata
one_step y x1 x2, generated(x1)
help one_step
```

If GitHub blocks the install (corporate firewall, etc.), clone the repo and use the local path version below.

### From a local clone

```stata
net install validmlinference, from("<path-to>/validmlinference-stata/") replace
```

### Development install (no copy to PLUS)

For iterating on the source without reinstalling each time, point Stata's adopath at the source tree directly:

```stata
adopath ++ "<path-to>/validmlinference-stata/src/ado"
adopath ++ "<path-to>/validmlinference-stata/src/mata"
adopath ++ "<path-to>/validmlinference-stata/src/help"
mata mata mlib index
```

### Uninstall

```stata
ado uninstall validmlinference
```

### Use inside another do-file (e.g., local projections)

Once installed, the commands are first-class Stata commands. Inside a local-projections script, the typical idiom is:

```stata
* Inside an LP loop: bias-corrected coefficient at horizon h
forvalues h = 0/`H' {
    quietly ols_bca y_lead`h' shock, generated(shock) fpr(0.05) m(1000)
    matrix B_lp[`=`h'+1', 1] = _b[shock]
    matrix B_se[`=`h'+1', 1] = _se[shock]
}
```

No re-initialisation of the Mata library is required between calls — `lvmli.mlib` stays loaded for the rest of the session.

## Commands

| Command | Purpose | Status |
|---|---|---|
| `ols` | OLS with HC0 standard errors and standardized coefficient ordering. | Implemented |
| `ols_bca` | Additive bias correction for a binary AI/ML-generated covariate. | Implemented |
| `ols_bcm` | Multiplicative bias correction for a binary AI/ML-generated covariate. | Implemented |
| `ols_bca_topic` | Additive bias correction for topic-model-generated regressors. | Implemented |
| `ols_bcm_topic` | Multiplicative bias correction for topic-model-generated regressors. | Implemented |
| `one_step` | Joint MLE of the regression and measurement-error model (normal / Laplace / Student-t residuals). | Implemented |
| `one_step_gmm` | Joint MLE with a `k`-component Gaussian mixture for the error term. | Implemented |

Run `help <cmd>` from inside Stata for the per-command reference.

## Quickstart

```stata
* Load the shipped remote-work example dataset (path relative to repo root
* once cloned; not needed if you just installed via net install and want to
* use your own data)
use "data/remote_work_data.dta", clear
gen log_salary = log(salary)

* OLS with HC0 robust SEs (Python parity)
ols log_salary wfh_rwo

* Bias-corrected (multiplicative): note the Stata option is `generated(...)`
* not `generated_var(...)` — Stata option names disallow underscores.
ols_bcm log_salary wfh_rwo, generated(wfh_rwo) fpr(0.009) m(1000)

* Topic-model bias correction: pass W (n x c), S (r x c), B (c x v) as Stata matrices
ols_bca_topic y q1 q2, wmatrix(W) smatrix(S) bmatrix(B) k(0.5)

* Joint MLE — Gaussian residuals (default), Laplace, or Student-t(df=5)
one_step log_salary wfh_rwo, generated(wfh_rwo)
one_step log_salary wfh_rwo, generated(wfh_rwo) dist(laplace)
one_step log_salary wfh_rwo, generated(wfh_rwo) dist(t) df(5)

* Joint MLE — Gaussian mixture outcome with multistart (k=2 components, 10 starts)
one_step_gmm log_salary wfh_rwo, generated(wfh_rwo) k(2) nguess(10) seed(0)
```

## Verification

The Stata port is checked against the Python package on a fixed test suite. Coefficients and standard errors must match within documented tolerances:

- Closed-form estimators (`ols`, `ols_bca`, `ols_bcm`, `ols_bca_topic`, `ols_bcm_topic`): tolerance ≤ 1e-8 on coefficients, ≤ 1e-7 on vcov. **All 7 closed-form fixture cases pass at machine precision (max |db| ≤ 7.7e-12, max |dV| ≤ 1.1e-18).**
- MLE estimators (`one_step`, `one_step_gmm`): looser tolerances (≤ 1e-3 / ≤ 1e-2), reflecting unavoidable differences between Mata's BFGS and Python's jaxopt LBFGS. **All four MLE fixtures pass:** `one_step` Gaussian at max |db| ≈ 1.21e-6 / max |dV| ≈ 1.08e-7; `one_step` Laplace at max |db| ≈ 2.94e-4 / max |dV| ≈ 2.10e-3; `one_step` Student-t(df=5) at max |db| ≈ 2.73e-6 / max |dV| ≈ 6.63e-9; `one_step_gmm` at max |db| ≈ 2.30e-4 / max |dV| ≈ 2.68e-6.

The full harness is **11/11 cases passing** as of 2026-05-26.

To run the harness (from the repo root):

```stata
do "tests/run_all_tests.do"
```

To regenerate the fixtures from the Python source (requires the upstream Python package and a Python 3.10+ env with `jax`, `jaxopt`, `numdifftools`, `pandas`, `scipy`, `patsy`):

```
python tests/refresh_fixtures.py
```

See `tests/README.md` for details on the fixture layout, precision conventions (JAX x64; `asdouble` on import), and per-case tolerances.
